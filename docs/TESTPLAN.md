# アカウント管理システム テスト計画

このテスト計画は、COBOLアプリの現行ビジネスロジックを網羅し、Node.jsアプリへの移行時にも活用できるよう設計されています。

| Test Case ID | Test Case Description                | Pre-conditions                | Test Steps                                                                 | Expected Result                                 | Actual Result | Status (Pass/Fail) | Comments           |
|--------------|--------------------------------------|-------------------------------|----------------------------------------------------------------------------|-------------------------------------------------|---------------|--------------------|--------------------|
| TC01         | 学生アカウント残高の表示            | 学生アカウントが存在する      | 1. メニューで「View Balance」を選択<br>2. 学生IDを入力                  | 学生アカウントの残高が正しく表示される         |               |                    |                   |
| TC02         | 学生アカウントへの入金（Credit）    | 学生アカウントが存在する      | 1. メニューで「Credit Account」を選択<br>2. 学生IDと金額を入力           | 残高が正しく加算され、確認メッセージが表示される |               |                    |                   |
| TC03         | 学生アカウントからの出金（Debit）   | 学生アカウントが存在し、残高十分| 1. メニューで「Debit Account」を選択<br>2. 学生IDと金額を入力            | 残高が正しく減算され、確認メッセージが表示される |               |                    |                   |
| TC04         | 残高不足時の出金エラー              | 学生アカウントが存在し、残高不足| 1. メニューで「Debit Account」を選択<br>2. 学生IDと過剰な金額を入力      | エラーメッセージが表示され、残高は変更されない   |               |                    |                   |
| TC05         | 学生アカウントの存在確認            | 学生アカウントが未登録         | 1. 任意の操作で未登録学生IDを入力               | 「アカウントが存在しない」旨のエラーが表示される |               |                    |                   |
| TC06         | アカウント追加時の必須項目検証      | なし                          | 1. 新規アカウント追加時に必須項目を未入力       | エラーメッセージが表示され、追加されない         |               |                    |                   |
| TC07         | アカウント削除                      | 学生アカウントが存在する      | 1. メニューで「Delete Account」を選択<br>2. 学生IDを入力                 | アカウントが削除され、確認メッセージが表示される |               |                    |                   |
| TC08         | アカウント更新                      | 学生アカウントが存在する      | 1. メニューで「Update Account」を選択<br>2. 学生IDと更新内容を入力        | アカウント情報が正しく更新される                |               |                    |                   |
| TC09         | 学生情報検索                        | 学生アカウントが複数存在する  | 1. メニューで「Search Account」を選択<br>2. 検索条件を入力                | 条件に合致するアカウント情報が表示される         |               |                    |                   |
| TC10         | 学籍番号フォーマット検証            | 新規/更新時                   | 1. 学籍番号を不正なフォーマットで入力           | エラーメッセージが表示され、処理されない         |               |                    |                   |

> このテスト計画は、ビジネスステークホルダーと検証を行い、Node.jsアプリのユニット・統合テスト作成時の基礎資料としてご利用ください。
